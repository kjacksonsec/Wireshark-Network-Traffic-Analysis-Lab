# Wireshark Network Traffic Analysis Lab

## Overview

In this lab, I used Wireshark to capture and analyze live network traffic on a Windows system. The objective was to gain hands-on experience with packet capture, protocol analysis, and network monitoring techniques commonly used by Security Operations Center (SOC) Analysts.

## Tools Used

- Wireshark

- Windows 11

- Command Prompt

- Web Browser

## Objectives

- Capture live network traffic

- Generate network activity

- Analyze common network protocols

- Apply Wireshark display filters

- Develop packet analysis skills

## Lab Procedure

### Step 1: Start Packet Capture

Opened Wireshark and selected the active Wi-Fi network adapter to begin capturing live network traffic.

### Step 2: Generate Network Activity

Generated network traffic by:

- Visiting example.com

- Running `ping google.com` in Command Prompt

### Step 3: Analyze Captured Traffic

Stopped the packet capture and reviewed the collected traffic using Wireshark display filters.

## Filters Used

### ARP

```text

arp

```

Purpose: Displays Address Resolution Protocol traffic used to resolve IP addresses to MAC addresses.

### TCP

```text

tcp

```

Purpose: Displays Transmission Control Protocol traffic used for reliable communication between systems.

### DNS

```text

dns

```

Purpose: Displays Domain Name System traffic used to resolve domain names into IP addresses.

### ICMP

```text

icmp

```

Purpose: Displays Internet Control Message Protocol traffic generated by ping commands.

## Findings

During the analysis, I observed:

- ARP traffic between local network devices

- TCP sessions used for network communication

- DNS requests generated while accessing websites

- ICMP traffic generated by ping activity

- Background network traffic occurring on the system

## SOC Analyst Relevance

Wireshark is commonly used by SOC Analysts to investigate suspicious traffic, validate alerts, troubleshoot network issues, and analyze communications between systems. This lab provided practical experience working with packet-level network data.
